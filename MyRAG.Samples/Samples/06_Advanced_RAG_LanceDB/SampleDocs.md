# RAG 範例文件

本目錄包含用於 RAG 測試的範例技術文件。

## 1. 關於 LanceDB 的介紹 (LanceDB_Intro.md)

LanceDB 是一個開源的向量資料庫，專為 AI 應用程式設計。它基於 Lance 格式，這是一種高性能的柱狀資料格式，旨在提供快速的隨機存取和高效的掃描性能。

LanceDB 的核心特點包括：
- **無伺服器架構**：LanceDB 可以直接嵌入到應用程式中，無需維護獨立的資料庫伺服器。
- **高性能向量搜尋**：支援快速的相似度搜尋，適用於大規模向量資料集。
- **整合資料管理**：不僅支援向量，還支援儲存結構化和非結構化資料。
- **磁碟優先設計**：即使資料量大於記憶體，也能保持高效能，這與許多僅限記憶體的向量資料庫不同。

## 2. 什麼是 Query Expansion (Query_Expansion.md)

查詢擴展 (Query Expansion) 是檢索增強生成 (RAG) 中的一種進階技術。其目的是透過生成原始使用者查詢的多個變體，來克服基於距離的相似度搜尋的局限性。

為什麼需要查詢擴展？
1. **解決用語差異**：使用者可能會使用與文件不同的詞彙（例如「電腦」 vs 「工作站」）。
2. **補充背景資訊**：簡單的查詢可能缺乏足夠的上下文來匹配相關文件。
3. **多視角檢索**：從不同的角度描述同一個問題，可以從向量空間中檢索到更全面的資料。

常見的實現方式是使用 LLM 接收原始查詢，並輸出 3-5 個語義相關但表述不同的新查詢。

## 3. 重疊切塊技術 (Overlap_Chunking.md)

在處理長文本時，我們必須將其切分成較小的區塊 (Chunks) 以適應模型的上下文窗口。重疊切塊 (Overlap Chunking) 指的是在相鄰的區塊之間保留一定比例的重複內容。

重疊切塊的優勢：
- **保持脈絡連貫性**：確保位於切分點附近的資訊不會因為被拆分到兩個區塊而遺失背景。
- **提高檢索精準度**：增加關鍵資訊出現在檢索結果中的機率，因為同一個資訊可能存在於多個重疊的區塊中。
- **改善回答品質**：提供給 LLM 的上下文更加完整，避免斷章取義。

通常建議的重疊比例為 10% 到 20%（例如 500 tokens 的區塊，保留 50 tokens 的重疊）。
